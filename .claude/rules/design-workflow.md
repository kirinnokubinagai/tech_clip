# デザインワークフロー規約

## 使用必須スキル

デザイン作業では以下のプロジェクトスキルを必ず呼ぶこと（`.claude/skills/` 配下に存在する）。

| フェーズ | スキル | タイミング |
|---|---|---|
| 要件整理・設計対話 | `ui-design-dialogue` | デザイン方針を決める前 |
| モックアップ・アイコン生成 | `image-gen` | 画像を生成するとき |
| UX品質レビュー | `ux-psychology-review` | モックアップ生成後・ユーザーレビュー前 |
| 大きな機能設計 | `brainstorming` | 新機能・画面設計の開始時 |

スキルを呼ばずにデザイン作業を開始することは禁止する。

---

## コミット・プッシュ禁止ルール（デザイン作業中）

デザイン作業中（モックアップ生成・spec 作成・UI画像）では以下を厳守する。

### 禁止事項

- ユーザーの明示的な「OK」「これでいい」「コミットして」なしに commit しない
- ユーザーが確認・承認していない画像ファイルを commit しない
- デザイン spec ファイルをユーザーレビュー前に commit しない

### 許可されるタイミング

commit してよいのは以下の条件をすべて満たすとき：

1. ユーザーがモックアップ・デザインを確認した
2. ユーザーが明示的に承認した（「いいね」「OK」「進めて」「コミットして」等）
3. 没になったファイルが含まれていない

### 適用範囲

- `docs/design/*.png`（モックアップ画像）
- `docs/superpowers/specs/*-design.md`（デザイン仕様）
- `apps/mobile/assets/images/*`（アプリ画像アセット）
- `apps/mobile/assets/icons/*`（アイコンアセット）

### 対象外（通常のコミットルールに従う）

- ソースコード（`.ts`, `.tsx` 等）
- インフラ設定
- テスト
