# UI設計スキル

## 概要
対話的にUI設計を進め、一貫性のあるデザインシステムを構築する。

## 設計プロセス

### Phase 1: 要件確認
1. 対象画面・機能の特定
2. ユーザーペルソナの確認
3. 主要なユースケースの洗い出し

### Phase 2: 情報設計
1. 情報階層の定義
2. ナビゲーション構造
3. コンテンツ優先度

### Phase 3: コンポーネント設計
1. Atomic Design原則に従った分解
   - Atoms: ボタン、入力、ラベル
   - Molecules: フォームフィールド、カード
   - Organisms: ヘッダー、フォーム全体
   - Templates: ページレイアウト
   - Pages: 実際の画面

### Phase 4: ビジュアルデザイン
1. カラーパレット（Primary/Secondary/Neutral/Semantic）
2. タイポグラフィスケール
3. スペーシングシステム（4px基準）
4. シャドウ・ボーダー

## デザイン原則（tech_clip固有）

### 色使い
- Primaryカラー: アプリのブランドカラー
- **AIっぽさを排除**: グラデーション、ネオンカラー禁止
- シンプルな単色ベース

### アイコン
- **Lucide Icons使用**（絵文字禁止）
- サイズ: インライン 16px、ボタン内 16-20px

### アニメーション
- 控えめなトランジション（150-300ms）
- prefers-reduced-motion対応必須

## 出力物
- コンポーネント仕様書
- Tailwind CSS クラス定義
- 実装用Reactコンポーネントコード
