# セキュリティ監査スキル

## 監査項目

### 1. 認証（Authentication）
- [ ] パスワードはbcryptでハッシュ化（コスト12以上）
- [ ] JWTトークンに適切な有効期限（access: 15分、refresh: 7日）
- [ ] HTTPOnly Cookieでトークン保存
- [ ] セッション固定攻撃対策

### 2. 認可（Authorization）
- [ ] リソース所有者チェック実装
- [ ] ロールベースアクセス制御（RBAC）
- [ ] 最小権限の原則

### 3. 入力バリデーション
- [ ] すべての入力をZodでバリデーション
- [ ] HTMLエスケープ処理
- [ ] ファイルアップロード制限（種類、サイズ）

### 4. SQLインジェクション対策
- [ ] ORM（Drizzle）使用
- [ ] 生SQLを使う場合はパラメータ化

### 5. XSS対策
- [ ] React自動エスケープ活用
- [ ] dangerouslySetInnerHTML使用時はDOMPurify

### 6. 機密情報管理
- [ ] 環境変数で管理
- [ ] .envは.gitignoreに追加
- [ ] ログに機密情報を出力しない
- [ ] ハードコードされた秘密がない

### 7. ネットワーク
- [ ] HTTPS強制
- [ ] CORS適切に設定
- [ ] レート制限実装

```markdown
- 中程度の問題: Y件
- 軽微な問題: Z件

### 脆弱性一覧
| 重大度 | カテゴリ | 問題 | ファイル | 修正方法 |
|--------|----------|------|----------|----------|
| 🔴 重大 | 認証 | ... | ... | ... |
| 🟡 中 | 入力検証 | ... | ... | ... |
| 🟢 軽微 | ... | ... | ... | ... |

### 推奨アクション
1. 即座に対応: ...

```

## 重大度定義

| レベル | 定義 | 対応期限 |
|--------|------|----------|
| 🔴 重大 | 攻撃可能、データ漏洩リスク | 即座 |
| 🟡 中 | 条件付きで悪用可能 | 1週間以内 |
| 🟢 軽微 | ベストプラクティス違反 | 次回リリース |
