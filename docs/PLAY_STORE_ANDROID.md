## 1. ストア掲載情報（メタデータ）

### アプリ名

```
TechClip
```

- 文字数制限: 50文字以内
- 日本語・英語ともに同じ名称を使用する

### 短い説明文（80文字以内）

```
技術記事・動画をAIが要約。エンジニアのための技術情報キュレーションアプリ。
```

### 詳細説明文（4000文字以内）

```
TechClipは、技術記事や動画をAIが自動で要約・翻訳してお届けする、エンジニアのための技術情報キュレーションアプリです。

【主な機能】

■ AI要約
最新の技術記事・動画を自動で要約。長い記事も短時間でポイントを把握できます。

■ 多言語対応
英語の技術コンテンツを日本語に翻訳。海外の最新情報もすぐに読めます。

■ パーソナライズドフィード
興味のある技術スタック・トピックを設定すると、あなたに合ったコンテンツが届きます。

■ オフライン閲覧
気になった記事を保存して、ネットワークのない環境でも閲覧できます。

■ 記事ブックマーク
後で読みたい記事をブックマーク。どのデバイスからでもアクセスできます。

【対象ユーザー】
- 忙しいエンジニア・開発者
- 技術トレンドをキャッチアップしたい方
- 英語の技術記事を読みたいが時間のない方
- 体系的に技術情報を管理したい方

【取り扱うコンテンツ】
- プログラミング（JavaScript・TypeScript・Python・Go・Rust など）
- クラウド・インフラ（AWS・GCP・Azure・Cloudflare など）
- AI・機械学習
- Web・モバイル開発
- OSS・ツール

最新の技術情報をスマートに収集し、エンジニアとしての成長を加速させましょう。

【プライバシー・セキュリティ】
- 収集するデータは最小限に限定
- データは暗号化して安全に保管
- 第三者へのデータ販売は行いません
- 詳細はプライバシーポリシーをご確認ください
```

### カテゴリ

- **メインカテゴリ**: ニュース & 雑誌
- **サブカテゴリ**: （なし）

### タグ（最大5個）

```
技術, プログラミング, AI, エンジニア, ニュース
```

## 2. スクリーンショット要件

### 必須サイズ

| デバイス種別 | 最小サイズ | 最大サイズ | 必須 |
|------------|-----------|-----------|------|
| スマートフォン（縦） | 320 x 568 px | 3840 x 7680 px | 必須（最低2枚） |
| スマートフォン（横） | 568 x 320 px | 7680 x 3840 px | 任意 |
| タブレット 7インチ | 600 x 1024 px | 1024 x 600 px | 推奨 |
| タブレット 10インチ | 1080 x 1920 px | 1920 x 1080 px | 推奨 |

**推奨解像度**: 1080 x 1920 px（FHD縦向き）

### スクリーンショット内容（推奨順序）

以下の画面を順番に用意する。各スクリーンショットには日本語のキャプションテキストを載せたデザインフレームを使用する。

| No | 画面 | キャプション例 |
|----|------|--------------|
| 1 | ホーム（記事フィード） | 「今日の技術トレンドを一覧で」 |
| 2 | AI要約表示 | 「長い記事もAIが3行で要約」 |
| 3 | 記事詳細 | 「日本語翻訳でスラスラ読める」 |
| 4 | ブックマーク一覧 | 「気になった記事をあとで読む」 |
| 5 | 検索・フィルター | 「興味の技術スタックで絞り込み」 |
| 6 | オフライン閲覧 | 「ネットなしでも既読記事を確認」 |
| 7 | 設定・プロフィール | 「自分好みにカスタマイズ」 |
| 8 | ダークモード対応 | 「目に優しいダークモード」 |

### Feature Graphic（必須）

- **サイズ**: 1024 x 500 px
- **形式**: PNG または JPEG
- **内容**: アプリロゴ + キャッチコピー（例: 「技術情報を、スマートに。」）
- アニメーション不可

### アプリアイコン（高解像度）

- **サイズ**: 512 x 512 px
- **形式**: PNG（透過なし）
- 角丸はGoogle Play側で自動適用されるため、正方形のまま作成する

### スクリーンショット作成手順

```bash
# Expo のシミュレーターでスクリーンショット取得
# Android エミュレーター起動後

# Android エミュレーターでスクリーンショット
adb shell screencap -p /sdcard/screenshot.png
adb pull /sdcard/screenshot.png ./screenshots/

# または EAS Build でデバイス実機から取得
eas build --platform android --profile preview
```

## 3. 署名鍵のセットアップ

### 概要

Android アプリの署名は EAS（Expo Application Services）を使用して管理する。
Google Play App Signing を有効にすることを推奨する。

### EAS での鍵管理（推奨）

```bash
# EASログイン
eas login

# 署名鍵の生成（初回のみ）
eas credentials

# 対話式メニューで以下を選択:
# Platform: Android
# Build profile: production
# Set up a new keystore
```

EAS が生成した鍵情報は EAS のサーバーに安全に保管される。

### 手動で Keystore を生成する場合

```bash
# Keystore ファイルの生成
keytool -genkey -v \
  -keystore techclip-release.keystore \
  -alias techclip \
  -keyalg RSA \
  -keysize 2048 \
  -validity 10000 \
  -storepass <KEYSTORE_PASSWORD> \
  -keypass <KEY_PASSWORD> \
  -dname "CN=TechClip, OU=Mobile, O=TechClip Inc, L=Tokyo, ST=Tokyo, C=JP"
```

### 鍵情報の保管ルール

- Keystore ファイルは Git にコミットしない（`.gitignore` に追加済み）
- パスワードは環境変数またはパスワードマネージャーで管理する
- 以下の情報を安全な場所（1Password 等）に保存する

```
ANDROID_KEYSTORE_PATH=<keystoreファイルのパス>
ANDROID_KEY_ALIAS=techclip
ANDROID_KEYSTORE_PASSWORD=<パスワード>
ANDROID_KEY_PASSWORD=<パスワード>
```

### EAS の eas.json 設定

```json
{
  "cli": {
    "version": ">= 3.0.0"
  },
  "build": {
    "development": {
      "developmentClient": true,
      "distribution": "internal"
    },
    "preview": {
      "distribution": "internal",
      "android": {
        "buildType": "apk"
      }
    },
    "production": {
      "android": {
        "buildType": "aab"
      }
    }
  },
  "submit": {
    "production": {
      "android": {
        "serviceAccountKeyPath": "./service-account.json",
        "track": "internal"
      }
    }
  }
}
```

### Google Play App Signing の有効化

初回アップロード時に Play Console 上で設定する。

1. Play Console > アプリを選択 > リリース > 設定 > アプリの署名
2. 「Google Play によるアプリ署名を使用する」を選択
3. EAS でビルドした AAB を最初のリリースとしてアップロードする

Google Play App Signing を有効にすることで:
- 署名鍵の紛失リスクを低減できる
- 鍵のローテーションが可能になる
- Play Protect との連携が強化される

### Google Play API サービスアカウント

EAS Submit での自動提出に使用する。

```bash
# 1. Google Cloud Console でサービスアカウントを作成
# https://console.cloud.google.com/

# 2. Play Console > 設定 > APIアクセス でサービスアカウントを招待
# 権限: リリース管理者

# 3. JSON鍵をダウンロードして保存
# ファイル名: service-account.json
# ※ Git にコミットしない

# 4. .gitignore に追加（設定済み確認）
grep "service-account.json" .gitignore
```

## 4. Play Console 設定

### 新規アプリ作成手順

1. [Google Play Console](https://play.google.com/console) にログイン
2. 「アプリを作成」をクリック
3. 以下の情報を入力する

| 項目 | 設定値 |
|------|--------|
| アプリ名 | TechClip |
| デフォルト言語 | 日本語 - ja |
| アプリの種類 | アプリ |
| 無料 / 有料 | 無料 |

### アプリのカテゴリ設定

Play Console > アプリ情報 > アプリカテゴリ

- **アプリの種類**: アプリ
- **カテゴリ**: ニュース & 雑誌

### ターゲットAPIレベル

Android の要件に従い、最新のターゲット API レベルを設定する。

```json
// app.json の android 設定
{
  "expo": {
    "android": {
      "compileSdkVersion": 34,
      "targetSdkVersion": 34,
      "minSdkVersion": 24,
      "package": "com.techclip.app"
    }
  }
}
```

- **compileSdkVersion**: 34（Android 14）
- **targetSdkVersion**: 34（Google Play の要件を毎年確認すること）
- **minSdkVersion**: 24（Android 7.0、市場の95%以上をカバー）

### データセーフティセクション

Play Console > アプリのコンテンツ > データセーフティ

以下の情報を申告する。

| データの種類 | 収集 | 共有 | 目的 |
|------------|------|------|------|
| メールアドレス | あり | なし | アカウント管理 |
| アプリのアクティビティ（閲覧履歴） | あり | なし | アプリ機能・パーソナライズ |
| クラッシュログ | あり | なし | アプリのパフォーマンス改善 |

申告フロー:

1. Play Console > データセーフティ
2. 「開始する」をクリック
3. 各データ種別で収集・共有・目的を回答する
4. 「保存」して審査に進む

### ストアの掲載情報設定

Play Console > アプリ情報 > メインのストア掲載情報

以下を設定する:

- [ ] アプリ名（50文字以内）
- [ ] 短い説明（80文字以内）
- [ ] 詳細説明（4000文字以内）
- [ ] アプリアイコン（512 x 512 px）
- [ ] Feature Graphic（1024 x 500 px）
- [ ] スクリーンショット（最低2枚）
- [ ] 連絡先メールアドレス
- [ ] プライバシーポリシーのURL

### リリーストラック

| トラック | 用途 | 対象者 |
|---------|------|--------|
| 内部テスト | 開発チームの動作確認 | 最大100人 |
| クローズドテスト（アルファ） | 限定ベータテスト | 招待ユーザー |
| オープンテスト（ベータ） | 公開ベータ | 希望者全員 |
| 本番 | 正式リリース | 全ユーザー |

**初回リリース推奨フロー**:
内部テスト → クローズドテスト → 本番（段階的ロールアウト10%から）

### 段階的ロールアウト設定

```
本番リリース時の推奨ロールアウト率:
1日目:  10%
3日目:  25%（問題なければ）
7日目:  50%
14日目: 100%
```

クラッシュレートが1%を超えた場合はロールアウトを停止し原因を調査する。

## 5. コンテンツレーティング

### レーティングアンケートの回答

Play Console > アプリのコンテンツ > コンテンツレーティング

以下の手順でアンケートに回答する:

1. 「アンケートを開始」をクリック
2. カテゴリを選択: **ニュース**
3. 各質問に回答する（TechClipの場合の回答例）

| 質問項目 | 回答 |
|---------|------|
| 暴力的コンテンツ | なし |
| 性的コンテンツ | なし |
| 差別的コンテンツ | なし |
| 薬物・アルコール | なし |
| ギャンブル | なし |
| ユーザー作成コンテンツ | あり（記事へのコメント機能実装時） |
| 位置情報の共有 | なし |
| 未成年のデジタル購入 | なし |

### 想定レーティング結果

- **対象年齢**: 全年齢対象
- **IARC評価**: Everyone / 全年齢

コメント機能などのUGC（ユーザー生成コンテンツ）機能を追加する場合は、
適切なモデレーション機能の実装とレーティングの再申告が必要。

## 6. 提出前レビューチェックリスト

### ビルド確認

```bash
# 本番用 AAB ビルド
eas build --platform android --profile production

# ビルド完了後、ダウンロードして実機確認
# Play Console 内部テストトラックにアップロードして動作確認
```

- [ ] EAS Build が正常に完了すること（AAB形式）
- [ ] 実機（Android 7.0 以上の複数デバイス）での動作確認が完了していること
- [ ] アプリのバージョンコード（`versionCode`）がインクリメントされていること
- [ ] アプリのバージョン名（`version`）が正しく設定されていること

### ストア掲載情報

- [ ] アプリ名が正しく設定されていること（50文字以内）
- [ ] 短い説明が日本語で記載されていること（80文字以内）
- [ ] 詳細説明が日本語で記載されていること（4000文字以内）
- [ ] アプリアイコンが設定されていること（512 x 512 px）
- [ ] Feature Graphic が設定されていること（1024 x 500 px）
- [ ] スクリーンショットが最低2枚設定されていること
- [ ] プライバシーポリシーのURLが設定されていること
- [ ] 連絡先メールアドレスが設定されていること

### 技術要件

- [ ] ターゲット API レベルが Google Play の要件を満たしていること（API 34以上）
- [ ] 64ビットアーキテクチャのサポートが含まれていること
- [ ] デバッグビルドではなくリリースビルドであること
- [ ] 署名が正しく設定されていること

### コンテンツ・ポリシー

- [ ] コンテンツレーティングのアンケートが完了していること
- [ ] データセーフティセクションが完了していること
- [ ] プライバシーポリシーが最新の状態であること（`docs/legal/` を確認）
- [ ] Google Play のデベロッパープログラムポリシーに準拠していること

### 機能動作確認

実機での動作確認項目（Android 実機 / エミュレーター）:

- [ ] アプリが正常に起動すること
- [ ] スプラッシュ画面が表示されること
- [ ] ユーザー登録フローが正常に動作すること
- [ ] ログイン・ログアウトが正常に動作すること
- [ ] 記事フィードが正常に表示されること
- [ ] AI要約が正常に表示されること
- [ ] 記事詳細画面が正常に表示されること
- [ ] ブックマーク機能が正常に動作すること
- [ ] オフライン時に保存済み記事が表示されること
- [ ] 検索機能が正常に動作すること
- [ ] 設定画面が正常に動作すること
- [ ] アカウント削除が正常に動作すること（GDPR対応）
- [ ] パスワードリセットが正常に動作すること
- [ ] バックボタンの動作が正常であること
- [ ] 画面回転時にクラッシュしないこと
- [ ] アプリがバックグラウンドから復帰したとき正常に動作すること

### セキュリティ確認

- [ ] API キー・シークレットがアプリバイナリに含まれていないこと
- [ ] HTTPS 通信のみ使用していること（HTTP 禁止）
- [ ] 個人情報がログに出力されていないこと
- [ ] ProGuard / R8 による難読化が有効になっていること（必要に応じて）

### アクセシビリティ確認

- [ ] コンテンツの説明（contentDescription）が主要な UI 要素に設定されていること
- [ ] フォントサイズを大きくした際にレイアウトが崩れないこと
- [ ] TalkBack（スクリーンリーダー）で主要な操作が可能であること

### 提出手順

```bash
# 1. 本番用 AAB ビルド
eas build --platform android --profile production

# 2. Play Console に手動でアップロード（または EAS Submit で自動提出）
eas submit --platform android --latest

# 3. Play Console でリリースノートを日本語で記載
# 4. 内部テストトラックで動作確認
# 5. 本番トラックに昇格（段階的ロールアウト 10% から）
```
